# ブログ管理フォルダ

ブログ運営先（会社・プラットフォーム）ごとにサブフォルダを分けて管理する。

```
blog/
  ameba/            ← アメーバブログ
    articles/        過去記事を蓄積（投稿したらここに保存）
    template.md       このブログ用の構成テンプレ（記事が増えるたびに更新）
    sidebar_tools/   サイドバー表示用HTMLツール（FXシミュレーター等）をバージョン管理
  <他のブログ名>/    ← 今後追加する場合は同じ構成で作る
    articles/
    template.md
```

サイドバー表示用ツールの管理ルールは [ameba/sidebar_tools/README.md](ameba/sidebar_tools/README.md) を参照。

## 運用ルール
1. 新しい記事を書いたら `<ブログ名>/articles/` に保存する。
2. 記事がある程度増えたら、過去記事の傾向（文体・構成・タグの付け方など）を見て
   `<ブログ名>/template.md` をより精度の高い内容に更新する。
3. 新しい記事を依頼するときは、必ずそのブログの `template.md` と
   `articles/` 内の過去記事を参照してから書く。
4. ブログを増やす場合は `blog/<新ブログ名>/articles/` `blog/<新ブログ名>/template.md` を作成する。

これにより、ブログごとに「そのブログらしい」構成・文体が記事を重ねるほど精度が上がっていく。
