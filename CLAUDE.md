# Claude エージェント設計書（Claude Code 運用用）

## 1. 返答スタイルエージェント
- 返答は常に「簡潔な日本語」。
- 結論を先に、補足は最小限。
- 冗長な説明・言い換えの繰り返しは禁止。

## 2. データ削除ポリシー
- 削除操作は即消去せず、必ず `/trash/` に移動。
- 完全削除はユーザーが「完全削除」と明示した場合のみ実行。

## 3. 質問整理・圧縮エージェント
- 無駄な質問・重複質問を自動統合。
- 目的・必要出力・不要要素を即整理し、最短ルートを提示。
- セッション消費を減らすため、質問を圧縮して提案。

## 4. AI作業場データ管理エージェント
- AI関連データは `/ai_workspace/` に集約。
- 推奨フォルダ構成：
  - `/ai_workspace/prompts/`
  - `/ai_workspace/templates/`
  - `/ai_workspace/logs/`
- 「◯◯テンプレを呼び出して」で即参照可能。
- 新規作成・整理するファイルは種類別の関連フォルダ（`blog/`, `data/`, `images/`, `html_tools/` など）に常時格納する。ルート直下に置きっぱなしにしない。
- 公開中ファイル（index.html、Netlify/GitHub Pages連携ファイルなど）は移動による影響を確認し、必要なら先にユーザーに確認する。

## 5. 予測・提案エージェント
- 過去の質問パターンを学習し、必要になりそうな形式を先回り提示。
- 例：
  - 「今回も前回の構成でまとめますか」
  - 「よく使うテンプレートを候補として提示します」

## 6. ブログ運営エージェント（複数ブログ対応・自己進化型）
- ブログは運営先ごとに `blog/<ブログ名>/` で管理（例：`blog/ameba/`）。
- 各ブログ内の `articles/` に投稿済み記事を蓄積し、`template.md` にそのブログ特有の構成・文体パターンを記録。
- 新しい記事を書く際は、対象ブログの `template.md` と `articles/` の過去記事を必ず参照してから生成する。
- 記事が増えるたびに過去記事の傾向を再抽出し、`template.md` を更新して精度を上げる（自己進化）。
- SEOを意識したタイトル案・タグ案を提示。
- 過去記事との重複テーマをチェックし、整理案を提示。
- 新しいブログ媒体を始める場合は `blog/<新ブログ名>/articles/` `blog/<新ブログ名>/template.md` を作成する。
- アメブロのサイドバー表示用HTMLツール（FXシミュレーター等）は `blog/ameba/sidebar_tools/<ツール名>/versions/` でバージョン管理する。編集・改良するたびに新バージョンとして追加保存し、上書きしない。公開状況は各ツールの `notes.md` に記録する。

## 7. Claude Code 運用ルール
- この md は Claude Code にそのまま貼り付けて使用する前提。
- 設定変更は md 内の該当セクションを編集するだけで反映。
- 新しいエージェント追加時はセクションを増やすだけで運用可能。

## 8. デプロイ運用ルール
- ツール・サイトの修正作業では、編集→動作確認（Playwright等）→ `notes.md` 記録、まで一通り終わらせてから最後にデプロイする。作業の途中で都度デプロイしない。
- 動作確認は常にClaudeのArtifact機能（`claude.ai/code/artifact/...`のURLを発行）で行う。外部依存のない単一HTMLツールは、ラッパータグ（`<!DOCTYPE>`/`<html>`/`<head>`/`<body>`）を除いた中身をArtifactとして公開し、そのURLをユーザーに渡して確認してもらう。同じファイルパスで再公開すれば同一URLのまま更新できる。Netlify等へのデプロイを確認手段として使わない。
- デプロイ（Netlify本番反映等、外部に公開される操作）は完全に許可制。実行する直前には必ず確認メッセージを出し、ユーザーの明示的な許可を得てから実行する（無許可でのデプロイは一切禁止。「進めて」等の一般的な指示だけではデプロイの許可とみなさない）。
